# Golden Runs

Golden runs calibrate the eval grader itself. They are not real agent benchmark
results.

The committed self-test is generated by `evals/graders/check_golden_runs.py`
instead of storing full workspace snapshots here. That keeps the repository
small and avoids stale copies of the harness. The script creates temporary
positive and negative run artifacts, invokes `harness_eval.py`, and asserts that:

- known-good artifacts pass;
- known-bad artifacts fail;
- important false-positive risks fail for the expected reason.

Run it with:

```bash
python3 evals/graders/check_golden_runs.py
```
